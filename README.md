# ❄️ NixOS Configuration (vijetasatyam)

This repository contains my personal, modular NixOS configuration. It is built to be **fully reproducible**, meaning this entire system can be rebuilt from scratch on any hardware using only these files.

## 🏗️ Philosophy: The Modular Approach

Standard NixOS setups often result in a single, massive `configuration.nix` that is difficult to navigate. This repository adopts a **modular architecture** where each aspect of the system is isolated into its own file. This makes debugging easier and allows for specific components to be swapped out without affecting the whole system.

### 📂 Module Breakdown

* **`configuration.nix`**: The central "brain." It orchestrates the system by importing all other modules and setting the `system.stateVersion`.
* **`system.nix`**: The foundation. Configures the bootloader (Systemd-boot/GRUB), system-wide environment variables, locales, and timezones.
* **`hardware-configuration.nix`**: The hardware "map." Generated by the system to handle drive mounting (UUIDs), CPU microcode, and kernel modules.
* **`networking.nix`**: Connectivity logic. Manages hostname, NetworkManager integration, and specific firewall ports (TCP/UDP).
* **`users.nix`**: Identity management. Defines my user profile, authorized SSH keys, and shell (Zsh/Bash) configurations.
* **`services.nix`**: Background daemons. Configures everything from the OpenSSH server and printing (CUPS) to the Display Manager (GDM/SDDM) and Window Manager.
* **`env-packages.nix`**: The stable software layer. Contains my daily-driver applications and system-wide CLI tools (Git, GPG, Neovim).
* **`unstable-packages.nix`**: The bleeding edge. A specialized module that pulls specific packages from the `nixos-unstable` channel when the latest features are required.

## 🚀 Management & Maintenance

### System Lifecycle Commands

| Action | Command |
| --- | --- |
| **Apply Changes** | `sudo nixos-rebuild switch` |
| **Test Config** | `sudo nixos-rebuild test` (Apply without saving to boot menu) |
| **Update Packages** | `sudo nix-channel --update && sudo nixos-rebuild switch` |
| **Cleanup** | `sudo nix-collect-garbage -d` (Removes old versions to save space) |
| **Check History** | `nixos-rebuild list-generations` |

### Disaster Recovery

If the system fails to boot after a change:

1. Reboot the machine.
2. Select a **Previous Generation** from the GRUB/Systemd-boot menu.
3. Once logged in, fix the error in the `.nix` files and run `switch` again.

## 🛠️ Installation on a New Machine

To replicate this setup on a fresh NixOS installation:

1. **Clone the repo:**
```bash
git clone https://codeberg.org/vijetasatyam/nixos-config.git ~/nixos-config
```


2. **Generate Hardware Config:**
Keep the `hardware-configuration.nix` generated by the installer on the new machine.
3. **Link and Build:**
```bash
sudo ln -sf ~/nixos-config/configuration.nix /etc/nixos/configuration.nix
sudo nixos-rebuild switch
```
